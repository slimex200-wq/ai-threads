# AI 카드뉴스 v2 설계

## 개요

기존 MVP(RSS → Claude API → PNG)에 뉴스 소스 확장, 2단계 필터링, GitHub Actions 자동화를 추가.

## 1. 뉴스 소스 확장

### 영문 추가 (3개)
- OpenAI Blog: `https://openai.com/blog/rss.xml`
- Google AI Blog: `https://blog.google/technology/ai/rss/`
- HuggingFace Blog: `https://huggingface.co/blog/feed.xml`

### 한국어 추가 (2개)
- AI타임스: `https://www.aitimes.com/rss/allArticle.xml`
- 지디넷코리아 AI: `https://zdnet.co.kr/rss/ai_news.xml`

기존 4개 + 신규 5개 = 총 9개 피드

## 2. 2단계 필터링

### 흐름
```
RSS 수집 (전체 기사)
  → 1차: 키워드 필터 (AI, LLM, GPT 등 30개 키워드)
  → 2차: Claude API 관련도 판별 (상위 10개 → 최종 4개 선별)
  → 카드 문구 생성
```

### 1차 키워드 필터
- `config.py`에 키워드 리스트 관리
- 제목 + summary에서 키워드 매칭 (대소문자 무시)
- 키워드 1개 이상 포함 시 통과

### 2차 Claude 필터 + 문구 생성
- 키워드 통과한 상위 10개 기사를 Claude에 전달
- Claude가 관련도 높은 4개 선별 + 카드 문구 생성을 한 번의 API 호출로 처리
- 비용 절감: 필터와 문구 생성 분리 X → 단일 호출

### 모듈
- `news_filter.py` — 키워드 필터링 로직
- `ai_writer.py` — 프롬프트 수정 (선별 + 문구 생성 통합)

## 3. GitHub Actions 자동화

### 워크플로우
- 파일: `.github/workflows/daily-cardnews.yml`
- 스케줄: 매일 오전 8시 KST (`cron: '0 23 * * *'` UTC)
- 수동 트리거: `workflow_dispatch` 지원

### 실행 단계
1. Python 환경 설정 + 의존성 설치
2. Chrome headless 설치 (html2image용)
3. `python cardnews.py` 실행
4. `output/` 폴더에 결과물 자동 커밋 + 푸시

### 시크릿
- `ANTHROPIC_API_KEY` → GitHub Secrets에 저장

## 4. 출력 경로 변경

- 로컬 실행: 기존대로 `~/Desktop/ai-cardnews/YYYY-MM-DD/`
- GitHub Actions: `output/YYYY-MM-DD/` (레포 내 자동 커밋)
- `--output` 옵션으로 오버라이드 가능

## 6. 에러 처리

- RSS 피드 일부 실패 → 나머지로 계속 진행
- 키워드 필터 후 기사 0개 → 키워드 없이 최신순 폴백
- Claude API 실패 → 에러 메시지 출력 후 종료
- GitHub Actions 실패 → Actions 로그로 확인
